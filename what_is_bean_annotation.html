What is @Bean Annotation

What is the use of @Bean Annotation

Earlier, we are using xml file for making the user defined and pre-defined classes as spring bean. But we should avoid coding practice using xml file for the configuration. Instead of this we can use 100% code driven technique where we are using the @configuration class for making the pre-defined class as spring bean and for making the user defined classes as spring bean we are using the @Component like annotations.

Lets take a simple Example to understand the use of @Bean Annotation

Here we are taking the simple class as MessageGenerator.java and giving annotation as @Component

Now consider the other class as AppConfig where we are giving the @configuration annotation, which internally uses the @Component annotation so this indicates that we can injest this class if we want.

The @Bean method of @configuration class will execute automatically and the default bean id will be method name. Here we have not mentioned any bean id explicitly so default bean id will be createTime in this case, but we can mention it like(@Bean(name="localDateTime")) likewise.

Now we can call the generateMessage() method from the MessageGenerator class using the MessageGenerator class object, then this will give the output as per our Local time.
